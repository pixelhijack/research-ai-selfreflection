## Part A — Higher frequency + use intensity

### Step 1 — DERIVE: recode ordinal and binary variables

```javascript
// ai_frequency comes in as a string label.
// For any comparison across groups, you need numbers.
// This is called "ordinal encoding" — you're preserving the order.

function encodeFrequency(r) {
  const map = {
    ai_frequency_daily: 5, // daily
    ai_frequency_weekly: 4, // several times a week
    ai_frequency_sometimes: 3, // sometimes
    ai_frequency_rarely: 2, // rarely
    ai_frequency_tried: 1, // tried but almost never
  };
  return map[r.ai_frequency] ?? null; // null = missing, exclude from analysis
}

// used_ai_psych: binary — did they use AI for psychological purposes?
// Check your actual string values, but likely something like:
function encodeUsedAiPsych(r) {
  const val = r.used_ai_psych;
  if (!val) return null;
  if (val.toLowerCase().includes("igen") || val.toLowerCase().includes("yes"))
    return 1;
  return 0; // "no" or "tried once" → 0
  // NOTE: "tried once" is ambiguous — decide whether that counts as yes or no
  // and document your decision. It affects your numbers.
}

// COMPOSITE: breadth of use — how many different problem types did they bring to AI?
// This is just summing the binary tool_goal_* flags.
// More flags selected = broader use = higher engagement.
const GOAL_KEYS = [
  "tool_goal_understanding", // emotions/feelings
  "tool_goal_decision", // decisions
  "tool_goal_organization", // daily life
  "tool_goal_self", // self-patterns
  "tool_goal_conversation", // hard conversations
  "tool_goal_learning", // learning psych topics
];

function breadthOfUse(r) {
  return GOAL_KEYS.reduce(
    (sum, key) => sum + (r["tool_goal"].includes(key) ? 1 : 0),
    0,
  );
  // returns 0–6, higher = broader use
}

// Apply to all respondents (excluding "no_share")
const ANALYSIS_GROUPS = [
  "diagnosed_nd",
  "suspected_nd",
  "neurotypical",
  "other_psych",
];

const enriched = respondents
  .filter((r) => ANALYSIS_GROUPS.includes(ndGroup(r))) // reuse ndGroup() from H1
  .map((r) => ({
    ...r,
    nd_group: ndGroup(r),
```

### Step 3 — TEST: is the group difference real or random noise?

This is where statistical testing comes in. You have two different tests for two different types of DV.

**For `ai_frequency` (ordinal 1–5 score): Kruskal-Wallis**

Conceptually, Kruskal-Wallis asks: _if I took all frequency scores and ranked them 1–126, would the ranks cluster by ND group, or would they be randomly mixed across groups?_ If ND respondents consistently hold the higher-ranked positions, the test returns a low p-value.

```javascript
// Kruskal-Wallis requires a library — use 'simple-statistics' in Node:
// npm install simple-statistics
const ss = require("simple-statistics");

// Build one array of scores per group
const freqByGroup = ANALYSIS_GROUPS.map((group) =>
  enriched
    .filter((r) => r.nd_group === group && r.freq_score !== null)
    .map((r) => r.freq_score),
);

// ss.kruskalWallis expects an array of arrays (one per group)
const kwResult = ss.kruskalWallis(freqByGroup);

console.log("Kruskal-Wallis H statistic:", kwResult);
// This gives you the H value. To get a p-value you need a chi-square
// table with (number_of_groups - 1) degrees of freedom,
// or use a p-value lookup library.
// For 4 groups: df = 3. If H > 7.81, p < 0.05 (significant).
// If H > 11.34, p < 0.01 (more significant).
```

**For `used_ai_psych` (binary yes/no): Chi-square**

Chi-square asks: _is the YES/NO distribution across ND groups different from what you'd expect if group membership had no effect?_ If ND groups show disproportionately high YES rates, p-value drops.

```javascript
// Chi-square for a 4-group × 2-outcome table
// You're building a contingency table:
//
//              YES    NO    (used_ai_psych)
// diagnosed_nd  A      B
// suspected_nd  C      D
// neurotypical  E      F
// other_psych   G      H

const contingencyTable = ANALYSIS_GROUPS.map((group) => {
  const members = enriched.filter(
    (r) => r.nd_group === group && r.used_psych !== null,
  );
  const yes = members.filter((r) => r.used_psych === 1).length;
  const no = members.filter((r) => r.used_psych === 0).length;
  return [yes, no];
});

// simple-statistics doesn't have chi-square built in for tables,
// but the manual calculation is understandable:

function chiSquare(table) {
  // table = [[a,b],[c,d],[e,f],[g,h]]
  const rowTotals = table.map((row) => row[0] + row[1]);
  const colTotals = [0, 1].map((col) =>
    table.reduce((s, row) => s + row[col], 0),
  );
  const grandTotal = rowTotals.reduce((a, b) => a + b, 0);

  let chiSq = 0;
  table.forEach((row, i) => {
    row.forEach((observed, j) => {
      const expected = (rowTotals[i] * colTotals[j]) / grandTotal;
      // "expected" = what you'd get if group had zero effect
      chiSq += Math.pow(observed - expected, 2) / expected;
    });
  });
  return chiSq;
  // df = (rows - 1) × (cols - 1) = 3 × 1 = 3
  // If chiSq > 7.81 → p < 0.05
  // If chiSq > 11.34 → p < 0.01
}

console.log("Chi-square:", chiSquare(contingencyTable));
```

**Important caveat before you run these tests:** with 126 respondents split into 4 ND groups, some cells may have fewer than 5 observations. Chi-square becomes unreliable when expected cell counts are below 5. If that happens, collapse `suspected_nd` + `diagnosed_nd` into one `"nd_any"` group and rerun as a 2×2 table. That's not cheating — it's standard practice, and you document it.

### Step 3 — The key comparison (the internal contradiction test)

You're not just comparing `set_rules` across groups — you're looking at the _ratio_ of engagement to self-regulation. This is what makes H2 part B interesting.

```javascript
// For each group, compute a simple engagement-to-regulation ratio.
// This is descriptive only — not a formal test — but it surfaces the pattern clearly.

ANALYSIS_GROUPS.forEach((group) => {
  const members = enrichedB.filter(
    (r) => r.nd_group === group && r.rules !== null,
  );

  const avgBreadth =
    members.reduce((s, r) => s + r.breadth, 0) / members.length;
  const avgRules =
    members.reduce((s, r) => s + r.rules.ordinal, 0) / members.length;

  // High ratio = high use relative to self-regulation = the vulnerable pattern
  // Low ratio = either low use, OR use with active boundaries
  const engagementToRegulation = avgBreadth / (avgRules + 0.1);
  // +0.1 avoids divide-by-zero if a group has avg rules near 0

  console.log({
    group,
    avg_breadth: avgBreadth.toFixed(2),
    avg_rules: avgRules.toFixed(2),
    engagement_regulation_ratio: engagementToRegulation.toFixed(2),
  });
});

// What to look for:
// diagnosed_nd → high avg_breadth, low avg_rules → highest ratio (most "at risk" pattern)
// neurotypical  → lower avg_breadth, higher avg_rules → lower ratio
```

Second, proceed with Part B (boundary-setting) regardless. It's independent of Part A and could still show a difference — high engagement with low self-regulation in ND respondents is a separate and arguably more interesting finding than just "higher use."
