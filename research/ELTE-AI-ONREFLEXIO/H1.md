## Step 1 — Build the composite variables

The idea is: instead of analyzing `tool_goal_understanding` and `tool_goal_self` separately, you collapse them into one signal. A respondent either shows "emotional use" or they don't.

```javascript
// For each respondent, derive composite flags.
// These are not in the raw data — you compute them.

respondents.forEach((r) => {
  // COMPOSITE: emotional disclosure use
  // "Did this person use AI for emotional/self understanding purposes?"
  // true if they selected EITHER of these options
  r.emotional_use = r.tool_goal_understanding === 1 || r.tool_goal_self === 1;

  // COMPOSITE: accessibility barrier as driver
  // "Did this person turn to AI because professional help was unavailable/slow?"
  r.accessibility_barrier =
    r.tool_motivation_professional === 1 || r.tool_motivation_time === 1;
});
```

This is where it gets interesting. You want to see if ND respondents show emotional use at a _different rate_ than NT respondents. A cross-tab is just a grouped count.

```javascript
// First, define your ND grouping
// (collapse the 5-level nd_status into the 4 analysis groups)
function ndGroup(r) {
  if (r.nd_status === "Diagnosztizált neurodivergens vagyok")
    return "diagnosed_nd";
  if (
    r.nd_status ===
    "Gyanítom, hogy neurodivergens lehetek, de nincs diagnózisom"

  if (r.nd_status === "Nem vagyok neurodivergens / nem tudok ilyesmiről")

}

// Group respondents, then for each group count emotional_use = true

groups.forEach((group) => {
```

## The general pattern you'll reuse for every hypothesis

Every analysis in this research follows the same three-step shape:

```
1. DERIVE   — compute composite or recoded variables from raw columns
2. COUNT    — get frequencies / percentages (descriptive)
3. GROUP    — split by your IV (nd_status / gender / is_professional / etc.)
              and compare rates across groups
```

The only thing that changes between hypotheses is which variables you derive in step 1 and which grouping variable you use in step 3. Statistical testing (chi-square, Spearman) is layered on top of step 3 to tell you whether the difference between groups is large enough to not be random noise — but conceptually it's always just "does this rate differ between these groups?"
