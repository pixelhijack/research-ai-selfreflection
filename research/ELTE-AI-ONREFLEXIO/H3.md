# H3a

## Step 1 — DERIVE

No new composite variables needed here. All four variables are already binary flags in your coded data. The only derivation is grouping nd_status, which you already have from H2.

```javascript
// The four cognitive offloading indicators
// The four cognitive offloading indicators
const OFFLOADING_KEYS = [
  { key: "tool_motivation_structure", label: "Wanted structured overview" },
  { key: "tool_usage_brainstorming", label: "Used as brainstorming partner" },
  {
    key: "tool_usage_secondary_opinion",
    label: "Used as neutral second opinion",
  },
  { key: "tool_goal_self", label: "Understanding own patterns" },
];

// Optional composite: did they endorse ANY offloading behavior?
// Useful for the cross-tab — one number per person instead of four
respondents.forEach((r) => {
  r.cognitive_offloading = OFFLOADING_KEYS.some((o) =>
    r.tool_motivation.includes(o.key),
  )
    ? 1
    : 0;

  // Also compute a score 0–4: how many offloading behaviors did they endorse?
  r.offloading_score = OFFLOADING_KEYS.reduce(
    (s, o) => s + (r.tool_motivation.includes(o.key) ? 1 : 0),
    0,
  );
});
```

## Step 2 — COUNT: overall endorsement rates

This is your baseline — across all 126 respondents, what % endorsed each offloading behavior? This answers H3a's core claim before even splitting by group.

```javascript
const total = respondents.length;

// Individual item frequencies
OFFLOADING_KEYS.forEach(({ key, label }) => {
  const count = respondents.filter((r) =>
    r.tool_motivation.includes(key),
  ).length;
  console.log({
    item: label,
    n: count,
    pct: ((count / total) * 100).toFixed(1),
  });
});

// Overall: what % endorsed at least one offloading behavior?
const anyOffloading = respondents.filter(
  (r) => r.cognitive_offloading === 1,
).length;
console.log({
  item: "Any cognitive offloading behavior",
  n: anyOffloading,
  pct: ((anyOffloading / total) * 100).toFixed(1),
});

// Average offloading score across all respondents
const avgScore =
  respondents.reduce((s, r) => s + r.offloading_score, 0) / total;
console.log("Mean offloading score (0–4):", avgScore.toFixed(2));
```

What to look for: if `tool_motivation_structure` and `tool_usage_secondary_opinion` rank highest, that's your core H3a confirmation — people are using AI primarily as a cognitive scaffold, not as an emotional crutch.

## Step 3 — GROUP: cross-tab with nd_status

```javascript
const ANALYSIS_GROUPS = [
  "diagnosed_nd",
  "suspected_nd",
  "neurotypical",
  "other_psych",
];

// Per group, per item — endorsement rate
ANALYSIS_GROUPS.forEach((group) => {
  const members = respondents.filter((r) => ndGroup(r) === group);

  const itemRates = OFFLOADING_KEYS.map(({ key, label }) => {
    const count = members.filter((r) => r.tool_motivation.includes(key)).length;
    return { label, pct: ((count / members.length) * 100).toFixed(1) };
  });

  const avgOffloadingScore =
    members.reduce((s, r) => s + r.offloading_score, 0) / members.length;

  console.log(
    JSON.stringify(
      {
        group,
        n: members.length,
        avg_offloading_score: avgOffloadingScore.toFixed(2),
        items: itemRates,
      },
      null,
      2,
    ),
  );
});
```

What to look for specifically for H3a: do `diagnosed_nd` and `suspected_nd` show higher rates on `tool_motivation_structure` in particular? The theoretical rationale was that ND users seek AI for structured output — so that item is your sharpest test of the hypothesis. The others are supportive context.

---

## One thing to watch for in your output

`tool_motivation_structure` is capped at max 3 selections per respondent (your survey said "choose max three" for the motivation question). This means the motivation items are slightly suppressed compared to the usage/goal items which had no cap. When you write up the frequencies, note this — it means you can't directly compare endorsement rates between motivation items and usage/goal items as if they were on the same scale.
